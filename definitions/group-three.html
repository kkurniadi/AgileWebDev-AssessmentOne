Content Definitions (Group Three)

Authentication is used by clients and servers to prove their identity to each other.
This is usually done by the server through use of a username and password, and by the client through verification of security certificates.
Web security refers to ways of protecting networks and computer systems from damage to or theft of software, hardware or data.
This includes protecting them from disruption of services they are expected to provide.

Hypertext transfer protocol is a protocol for transmitting hypermedia documents, such as HTML.
It follows a client-server model, in which the client opens a connection to make a request, then waits until it receives a response.
These requests and responses contain different types of information, such as a URL, HTTP method, request headers and a body.

Session management is the process of securely handling sessions, or series of HTTP requests and transactions to web-based applications and services.
These sessions are initiated by a single user or entity, who often authenticates their identity using a password or other authentication protocol.
As session management involves the sharing of secrets with authenticated users, the security of cryptographic network communications will thus also be paramount to its security.

Stateless programming is when programs do not retain information from the last session in the next one.
This includes user settings, configuration settings, transaction information, etc.
The HTTP protocol is a good example of stateless programming, as connections are closed after a web page is delivered to a user and the server does not retain any data between requests.
